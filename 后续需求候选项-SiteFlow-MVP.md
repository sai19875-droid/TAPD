# SiteFlow 后续需求候选项（需求池 / Roadmap）

> 场景：阶段一需求池整理 / 阶段六后续行动项。基于 PRD v1.5「不在本期范围」与已知限制梳理。
> 目的：为 MVP 之后的版本规划提供候选清单与优先级建议。

## P1（建议下一版本核心）
| 候选 | 价值 | 复杂度 | 备注 |
|------|------|--------|------|
| **多币种 + 实时汇率换算** | 支撑非美元市场，国际化前提 | 中 | 订单 `currency` 当前跟随主站（USD），需汇率源 + 换算口径 |
| **佣金结算 / 提现** | 完成财务闭环 | 高 | MVP 仅预估+标记可结算；需账户/打款/对账 |
| **完整自由画布建站** | 提升建站灵活度 | 高 | M5 仅区块级拖拽；P1 做像素级自由布局 |
| **云仓模块** | 履约能力 | 高 | 仓储/物流，跨境标配 |
| **中东 RTL 深度定制** | 中东市场适配 | 中 | 当前仅基础 RTL 渲染 |
| **GDPR 合规** | 欧盟市场准入 | 中 | 数据主体权利/同意管理 |

## P2（增强 / 探索）
| 候选 | 价值 | 复杂度 | 备注 |
|------|------|--------|------|
| 三级分销裂变 | 增长杠杆 | 中 | PRD 列为 P2 |
| 设计稿协同 / Figma 集成 | 设计-研发对齐 | 低 | 阶段三评审素材流转 |
| 数据看板增强 | 经营洞察 | 中 | 当前 analytics 已接真实数据 |
| 自动化回归 CI | 质量保障 | 低 | 现有 3 脚本可接入流水线 |
| 多语言 SEO 增强 | 获客 | 中 | sitemap/hreflang |

## 优先级建议
1. **先做 P1 的「多币种 + 佣金结算」**——直接补齐 MVP 最显眼的财务/国际化缺口，是规模化获客的前提。
2. **自由画布** 视内测反馈：若种子用户强烈需要更高建站自由度，提前；否则后置。
3. **云仓 / GDPR** 与具体市场拓展绑定，按目标市场节奏排期，不抢跑。

## 待确认问题（转干系人）
- 下一版本目标市场是否含非美元区？（决定多币种优先级）
- 佣金结算是否需对接具体支付/打款通道？（决定 P1 范围）
- 内测用户中是否有中东/RTL 强需求？（决定 RTL 深度定制排期）
