# SiteFlow MVP 种子用户内测指南（v1.5）

> 配套文档：`PRD-核心闭环-MVP.md`（v1.5）、`验收清单-SiteFlow-MVP-v1.5.md`
> 目标：在「建站 + 分销」核心闭环跑通后，招募 3–5 个种子用户完成真实场景走查，回收转化与佣金结算反馈。
> 当前阶段：研发已完成 M1–M5 + B2C 购物车/合并结算 + B2C 税费/运费 + 双后端收敛；TAPD 事项追踪与项目资料库归档待连接器授权后补齐。

## 一、环境与启动

| 服务 | 启动 | 地址 |
|------|------|------|
| Web（在路线径 / storefront / dashboard） | `cd apps/web && npm run dev` | http://localhost:3000 |
| API（NestJS，只读聚合等） | `cd apps/api && npm run start:dev` | http://localhost:3001/api/v1 |

- 数据库：PostgreSQL（`siteflow`），MVP 期数据真实落库（非 mock）。
- 演示主站：`site-demo-1`（含 TECH-001 / TECH-002 等商品）；演示代理站：由 Demo Agent 绑定，域名 `agent-*.siteflow.io`。

## 二、两类角色与走查路径

### 角色 A：主账号（建站方 / 品牌方）
1. **建站**：建站向导选模板 → 选语言 → 自动生成多语言页面。
2. **设批发价**：为商品配置价格等级（tier），设定代理商佣金规则（如 15%）。
3. **生成代理站**：为代理商一键生成独立域名站点，继承主站商品 + 该代理商 tier 价格。
4. **看汇总**：dashboard / `GET /api/orders` 查看代理站订单回流、营收、按状态分布、待结算佣金。

### 角色 B：代理商（分销方）
1. 访问独立域名站点，浏览多语言商品（RTL 语言如阿拉伯语自动右到左排版）。
2. **加购**：商品列表 / 详情页「加入购物车」，购物车跨页持久化（localStorage）。
3. **合并结算**：购物车多商品一次性结算，订单经 `POST /api/orders` 回流主账号。
4. 下单后主账号侧即时出现订单与佣金预估。

## 三、重点验证项（建议种子用户覆盖）

- **B2C 购物车 + 合并结算**：多商品加购、去重累加、刷新不丢、合并下单。
- **B2C 税费/运费（本次新增）**：
  - 单笔小计 **≥ $200 免运费**；**< $200 收 $9.99 运费**。
  - 税费 = 小计 × **8%**（仅对商品计，不含运费）。
  - 合计 = 小计 + 运费 + 税费；**佣金按含税含运费的总额计提**。
  - 订单详情 / 列表均可看到「小计 / 运费 / 税费 / 合计」分项。
- **多语言与 RTL**：至少验证 EN / ZH / AR（AR 应 `dir=rtl`）。
- **订单回流与佣金**：代理商下单 → 主账号汇总出现 → 佣金预估正确（按总额）。
- **异常路径**：上报含未知 SKU 的订单 → 标记 `unmatched`，不计入佣金，待人工核对。

## 四、已知限制（向内测用户透明说明）

- **完整自由画布拖拽**留 P1（本期为区块级轻量拖拽，非任意坐标布局）。
- **多币种**仅跟随主站商品币种（seed=USD），MVP 不做实时汇率换算。
- **TAPD 事项追踪 / 项目资料库 PRD 归档**待外部连接器授权后补齐，本期内测反馈先走本指南的反馈模板。
- **佣金结算/提现**仅做预估 + 标记可结算，实际打款留 P1。
