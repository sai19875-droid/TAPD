# SiteFlow MVP v1.5 内测启动清单

> 用途：内测（beta）启动的可执行 Checklist，配合 `种子用户内测指南.md` 使用。
> 前提：GitHub 事项 #1（B2C 税费/运费）、#2（双后端收敛）已完成并验证；交付物已归档。

## 一、启动前（Pre-launch）
- [x] 核心闭环端到端验证通过（建站→代理站→订单回流→佣金预估）
- [x] 质量门禁全绿（web/api `tsc` 0 errors；计费 7/7、结算 8/8、seed 无缺口）
- [x] 交付物归档（PRD/验收/内测指南/Release Notes/风险清单/进度摘要，见 GitHub `sai19875-droid/TAPD`）
- [x] 跟踪事项就绪（GitHub #1/#2/#3，含验收标准与依赖；标签已打）
- [ ] issue 关注人到位（技术/测试/设计，待提供 handle）
- [ ] 演示环境 `site-demo-1` + Demo Agent 可用，种子数据校验通过

## 二、招募与发放
- [ ] 确定 A 类（主账号视角）与 B 类（终端顾客视角）内测用户各若干
- [ ] 发放 `种子用户内测指南.md`，明确角色任务与反馈模板
- [ ] 在 issue #3 评论区开通反馈入口，或建专属反馈 issue

## 三、内测执行（Execution）
- [ ] A 类：建站 → 设批发价 → 生成代理站 → 查看订单回流与佣金预估
- [ ] B 类：代理站加购 → 合并结算（核对税费/运费）→ 完成下单
- [ ] 重点验证：免运阈值 $200、税率 8%、佣金按总额计提、SKU 不匹配 `unmatched` 降级
- [ ] 监控多语言 / RTL 前台渲染与 SEO（SSR 直出）稳定性

## 四、回收与闭环
- [ ] 按反馈模板回收体验与阻断性 Bug
- [ ] 阻断性 Bug 建 GitHub issue 并指派（关联回 #3 或对应模块事项）
- [ ] 每周一次内测进度小结（见 `研发进度摘要-SiteFlow-MVP-v1.5.md` 模板）
- [ ] 内测达标后进入「上线复盘」（阶段六）

## 五、退出标准（Exit Criteria）
- [ ] 内测用户跑通主路径比例 ≥ 预期（PRD 北极星：闭环完成率 MVP ≥ 30%）
- [ ] 阻断性 Bug 全部建项并进入处理
- [ ] 税费/运费计费在真实场景零差错报告
- [ ] 订单回流与主账号汇总一致性无异常
